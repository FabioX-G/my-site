What does "Internet of Things (IoT)" mean?

The Internet of Things (IoT) describes the network of physical objects—“things”—that are embedded with sensors, software, and other technologies for the purpose of connecting and exchanging data with other devices and systems over the internet. These devices range from ordinary household objects to sophisticated industrial tools. [1]

What is an IoT platform?

An IoT platform bridges the gap between device sensors and data networks. It provides an insight into the data used in the backend application. An IoT platform is a set of components that allows developers to spread out the applications, remotely collect data, secure connectivity, and execute sensor management. [2]

What are the different types of IoT devices?

  • Actuators 制动器；传动装置: perform physical actions when their control centers gives instructions, usually in response to changes identified by sensors. They’re a type of transducer.

  • Transducers 换能器；变频器；转能器: convert one form of energy into another

  • Embedded systems 嵌入式系统: microprocessor-based or microcontroller-based systems that manage a specific function within a larger system. They include both hardware and software components.

  • Intelligent devices 智能设备: Devices that have the ability to compute. They often include a microcontroller and may utilize services.

  • Microcontroller unit (MCU) 微控制器单元: These small computers are embedded on microchips and contain CPUs, RAM, and ROM.

  • Microprocessor unit (MPU) 微处理器单元: perform the functions of CPUs on single or multiple integrated circuits

  • Non-computing devices 非计算设备: only connect and transmit data and do not have the ability to compute

  • Sensors 传感器；感应器: detect changes in their environments and create electrical impulses to communicate

What tasks does an IoT device/product need to perform?

  1. Acquire real-world data via sensors

  2. Analyze data locally (edge computing)

  3. Connect to the cloud to transmit data and receive commands

  4. Store data in the cloud

  5. Analyze data in the cloud to create insights

  6. Command the “things” to perform specific tasks based on insights

  7. Present insights to users

Additionally, it is also important to be able to:

  1. Perform all operations securely across the IoT technology stack

  2. Identify and manage all of your IoT devices (at scale)

How IoT devices communicate with the network?

IoT devices communicate using IoT protocols. [3]

Internet protocol (IP) is a set of rules that dictates how data gets sent to the internet. IoT protocols ensure that information from one device or sensor gets read and understood by another device, a gateway, a service. Different IoT protocols have been designed and optimized for different scenarios and usage. [3]

What does "protocol" mean?

A protocol is a set of rules and guidelines for communicating data. Rules are defined for each step and process during communication between two or more computers. Networks have to follow these rules to successfully transmit data. [6]

What is the main problem with current IoT progress?

Smart homes have historically been plagued by a lack of interoperability between devices. 

To address this issue, the Connectivity Standards Alliance (CSA), along with industry leaders such as Google, Apple, and Samsung, has developed an IoT protocol focused on interoperability: Matter. [7]

The game-changing protocol "Matter" is here. What is it? What makes it different?

Matter is an open wireless connectivity protocol designed to create interoperability between IoT devices, specifically targeting the smart home. Developed by a consortium of 280 industry leaders, including Apple, Samsung, IKEA, Google, Huawei, Schneider Electric, and Amazon, Matter aims to create a local network for direct device-to-device communications in which all Matter-certified devices can operate. [7]

Matter leverages connectivity via IP protocols. Specifically, Matter relies heavily on Wi-Fi for high-bandwidth applications and leverages Thread for low-bandwidth, ultra-low-power IoT applications by creating a mesh network within the home. Other supported protocols include ZigBee and Z-Wave. [7]

Why is Thread better than other existing similar networks?

Unlike other low power mesh networks such as Zigbee, Z-Wave, and Insteon, Thread doesn't need an extra hub or gateway device to translate the protocols to communicate with the IP-connected world.

This is because Thread uses the same IP protocols as Ethernet and WiFi. Other low power mesh networks do not use IP protocols and require "protocol translation" by a hub or gateway to communicate with the IP-connected world. Therefore, Thread is the preferred low power mesh network.

What is a "mesh network"? 

A mesh network is a group of devices that act as a single Wi-Fi network; so there are multiple sources of Wi-Fi around your house, instead of just a single router. These additional Wi-Fi sources are called points. [9]

The real name of Mesh Wifi is called the IEEE 802.11s wireless standard defines how specific Wi-Fi devices (referred to as nodes) can be interconnected to create a WLAN mesh network. [13]

What is the difference between "mesh wifi network", "wifi range extender", and "wifi repeaters"?

  • The main difference between a traditional WiFi extenders and mesh WiFi is that mesh WiFi creates a single network, so you’ll have one Wi-Fi name and password throughout your home allowing seamless transition as you move around plus mesh offers better performance as they leverage one another’s WiFi signals to deliver the best efficiency instead of working independently. [11]

Wifi range extender and wifi repeaters are just two types of wifi boosters:

  • WiFi Repeaters repeat signals from the main router without creating a separate Wi-Fi network, but rather increase the coverage of the existing network. [12]

  • Range extenders, on the other hand, create their own distinct WiFi network. Thus, client devices would detect different WiFi networks for the main router and each of its extenders. [12]

Why are these network types important to IoT devices?

Different IoT devices might work better with different internet connection types. Although WiFi is great for cameras, thermostats, and other plug-in devices, Thread is better for small, battery-powered things.
